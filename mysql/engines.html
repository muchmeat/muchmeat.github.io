# 存储引擎

# 查看当前数据库支持的存储引擎

show engines;

# 指定引擎（表类型）

CREATE TABLE my_MyISAM
(
    id   bigint NOT NULL COMMENT 'ID',
    name varchar(255) COMMENT '用户名'
) ENGINE = MyISAM COMMENT ='使用MyISAM存储引擎';

CREATE TABLE my_MEMORY
(
    id   bigint NOT NULL COMMENT 'ID',
    name varchar(255) COMMENT '用户名'
) ENGINE = MEMORY COMMENT ='使用MEMORY存储引擎';

# InnoDB

  • 介绍

    是一种兼顾高可靠性和高性能的通用存储引擎，在MySQL5.5之后作为默认存储引擎

  • 特点

    DML操作遵循ACID模型，支持事务

    行级锁，提高并发访问性

    支持外键FOREIGN KEY约束，保证数据的完整性和正确性

  • 文件

    xxx.ibd:xxx代表的是表名。使用InnoDB引擎的表都会对应这样的表空间文件，存储该表的表结构（frm、sdi）、数据和索引。

    其中参数：innodb_file_per_table 决定一个表是否对应一个ibd,可使用以下命令查看开关状态

show VARIABLES like 'innodb_file_per_table';

# 关于idb

该文件在windows系统中一般是在 C:\ProgramData\MySQL\MySQL Server 8.0\Data下。 查看具体数据库内某张表idb中的sdi信息，比如数据库是sys,表是sys_user,则如下执行

cd C:\ProgramData\MySQL\MySQL Server 8.0\Data\sys
ibd2sdi sys_user.ibd

# MyISAM

  • 介绍

    MySQL早期的默认存储引擎

  • 特点

    不支持事务，不支持外键

    支持表锁，不支持行级锁

    访问速度快

  • 文件

    xxx.sdi:存储表结构信息

    xxx.MYD:存储数据

    xxx.MYI:存储索引

# Memory

  • 介绍

    数据是存储在内存中，由于受到硬件问题，或断电问题的影响，只能将这些作为临时表或缓存使用

  • 特点

    内存存放

    hash索引（默认）

  • 文件

    xxx.sdi:存储表结构信息
