# maven 相关

# springboot maven 资源问题

当项目中存在静态资源，比如一些excel、word模板，我们可能使用a标签的 href 来通过url直接访问下载，或者调用一个下载接口来访问。在maven中，默认情况下会对resource里面的资源文件进行压缩，压缩可能导致文件不可用。这时就需要加上maven相关的插件

# 1.excel


<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-resources-plugin</artifactId>
    <version>2.6</version>
    <configuration>
        <nonFilteredFileExtensions>
            <!-- 需要过滤掉不需要编码的文件：过滤后缀为.xlsx或者.xls的所有文件，不对其进行统一编码-->
            <nonFilteredFileExtension>xlsx</nonFilteredFileExtension>
            <nonFilteredFileExtension>xls</nonFilteredFileExtension>
        </nonFilteredFileExtensions>
    </configuration>
</plugin>

# 2.woff woff2 ttf

编译完后，清除浏览器缓存


<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-resources-plugin</artifactId>
    <version>2.6</version>
    <configuration>
        <nonFilteredFileExtensions>
            <!-- 过滤后缀为.woff或者.woff2或.tff的所有文件，不对其进行统一编码 -->
            <nonFilteredFileExtension>woff</nonFilteredFileExtension>
            <nonFilteredFileExtension>woff2</nonFilteredFileExtension>
            <nonFilteredFileExtension>ttf</nonFilteredFileExtension>
        </nonFilteredFileExtensions>
    </configuration>
</plugin>

# MAVEN 离线

1.将maven仓库拷贝到目标机器 {path}

2.settings.xml配置


<localRepository>{path}</localRepository>

<offline>true</offline>

<mirror>
<id>central</id>
<mirrorOf>*</mirrorOf>
<name>central</name>
<url>file://{path}</url>
</mirror>

其中的 {path} 就是本地仓库路径

3.清理所有 _remote.repositories 文件

  • windows
cd D://maven
for /r %i in (_remote.repositories) do del %i
  • linux
cd D://maven
find ./repository -name "_remote.repositories" -exec rm {} \;
